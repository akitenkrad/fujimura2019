# アーキテクチャ

[English](architecture.md) | [日本語]

藤村・日野 (2019) の構造方程式モデルを socsim の `WorldState` へ翻訳し，固定 6 フェーズループ上の 8 つの `Mechanism` で進める．

## `SilenceWorld`

従業員は Watts–Strogatz 組織ネットワーク（`socsim-net::SocialNetwork`）上に存在する．空間グリッドは使わない．

```
SilenceWorld {
  clock, employees: BTreeMap<AgentId, Employee>, teams: Vec<Team>,
  network: SocialNetwork, issue_salience σ(t), climate_of_silence C(t),
  locale: {JaJp | EnUs}, hierarchy_strength L, eta η, retaliation_this_step,
}
Employee {
  level, team, tenure, psych_safety ψ, fear f（怖れ）, acquiescent a（黙従）,
  ivt_strength ι, expression, silence_motive: Option<Motive>, voice_threshold θ,
  voiced, silenced,   // 独立した 2 つの行動チャネル（原著 H5）
}
Motive ∈ { Acquiescent, Quiescent, Prosocial, Opportunistic }
```

`BTreeMap` のキーが決定論的なソート済み活性化順序を与える．

### 独立した発言／沈黙チャネル

原著は沈黙 ⊥ 発言（r=.02）を見いだす：両者は対立ではなく別個の行動次元である．そこでモデルは単一の排他的 expression ではなく，独立な tick 単位フラグ `voiced`（`voice_decision` が設定）と `silenced`（`silence_spiral` が設定）を記録する．`expression` は表示用要約（voiced なら Voice，silenced なら Silence，それ以外 Neutral）であり，パネル CSV は両フラグを別列で記録する．

## 初期分布（原著表 1，[0,1] に再スケール）

潜在状態を clip 付き正規分布から IID サンプル：ψ N(.613, .139)，fear N(.450, .214)，acquiescent N(.457, .197)，ι N(.55 JP / .40 EN, .15)，θ N(.40, .15)．

## 8 メカニズム × 6 フェーズ

| # | Mechanism | Phase | SEM パス |
|---|-----------|-------|----------|
| 1 | `issue_salience` | Environment | σ(t) 平均回帰 + 任意ショック |
| 2 | `retaliation_event` | Environment | 確率的報復 → `retaliation_this_step` |
| 3 | `fear_appraisal` | Decision | **ψ → 怖れ**（原著 β=−.54）：`f ← f + α_f(g_f − f)`，ψ 低下で g_f 上昇 |
| 4 | `voice_decision[_rule]` | Decision | **怖れ → 発言**（β=−.23）：LLM（JP/EN プロンプト）or ロジスティック ablation；`voiced` を設定 |
| 5 | `acquiescent_update` | Interaction | **怖れ → 黙従**（β=+.86）：`a ← (1-α_a)a + α_a σ_sat(γ_fa·f + ι)` |
| 6 | `silence_spiral` | Interaction | **黙従 → 沈黙**（β=+.52）：`P(silence) = σ(β_a·a + β_C·(ρ−anchor) − β_θ·protect)`；`silenced` + motive を設定 |
| 7 | `psafety_update` | PostStep | 報復なしの発言が ψ を緩やかに上げる |
| 8 | `climate_silence` | PostStep | 次 tick 用に C(t) を集計 |

全メカニズムは同期更新（snapshot → batch write）．LLM 呼び出しは `voice_decision` に限定し，他は決定論的．

## RNG ストリーム

`derive_seed(root, &[label])` で独立ストリームを派生：`0` 世界初期化（潜在状態サンプリング），`1` エンジン（scheduler / ベルヌーイ抽選），`2` ネットワーク生成，`3` LLM `(agent_id, t)` シード．ルールモードは bit 決定論的，LLM モードはプロンプトキャッシュで擬似決定論的．

## β̃ 推定

4 つの SEM パス係数は Rust 側に**ハードコードしない**．Python の `fit-sem` が `agent_panel.csv` を読み，エージェントごとに 1 行の横断観測（t=0 を除く潜在状態の時間平均 + 行動頻度）を構成し，semopy で SEM をフィットして標準化 β̃・Wald CI・CFI/GFI/RMSEA を報告する．
